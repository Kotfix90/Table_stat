# Прототип детекции уборки столов

Данный проект реализует прототип системы для анализа времени занятости столиков в кафе/ресторанах на основе видеопотока. Используется модель YOLOv8 для детекции людей и логика состояний для определения моментов подхода к столику, его занятию и ухода гостей.

4. **Логика детекции событий**

    В основе логики лежат несколько ключевых переменных:
    - **person_in_zone** — показывает, есть ли человек в зоне столика (`True` или `False`).
    - **current_state** — текущее состояние столика (например, пуст, занят и т.д.).
    - **empty_counter** — счётчик кадров, в которых в зоне не было людей.
    - **entry_time** — время, когда гость подошёл к столику.
    - **last_exit_timestamp** — время, когда гость покинул столик.
    - **cleaning_delays** — список временных интервалов между уходом одного гостя и подходом другого.
    - **events_list** — журнал событий для логирования.

    **Логика работы событий:**

    Когда в зоне появляется человек (`person_in_zone == True`), счётчик `empty_counter` обнуляется. Если столик был пуст (`STATE_EMPTY`), фиксируется время подхода (`entry_time`) и логируется событие **"Approach"**. Если до этого кто-то уходил, рассчитывается задержка между уходом и новым подходом (`delay = curr_sec - last_exit_timestamp`).

    Если столик находится в состоянии **"Approach"** и человек остаётся в зоне дольше 2 секунд, состояние меняется на **"Occupied"**, и это событие тоже логируется.

    Когда в зоне никого нет (`person_in_zone == False`), счётчик `empty_counter` увеличивается. Если он достигает 25 кадров (`EMPTY_CONFIRM_FRAMES`), столик переходит в состояние **"Empty"**, фиксируется время ухода (`last_exit_timestamp`), и событие логируется.

5. **Результаты:**
    1. В течении работы в консоль выводится промежуточная статистика времени между уходом и подходом
    2. Так же статистика приврдится в файлах report.txt и логированных данных в events_log.csv
